Module 3 — Vibe Coding 101

Describe it. Let the AI build it. Run → break → fix.

Daryl Roberts · obney.ai

What is vibe coding?

Building software by describing what you want in plain English. You stay the architect; the AI is a very fast junior who never sleeps.
⚠️ Bust the myth: it's NOT "generate code and ignore it." You read what comes back — the code is still the battleground.

Model vs. agent

Claude = the brain

It thinks and talks. That's chatting.

Claude Code = the brain with hands

It edits files and runs things on your machine. That's vibe coding.

Plan mode = look before you leap

The AI proposes a plan, you approve it, THEN it changes anything. Course-correcting a plan is cheap; undoing a mess is expensive.

Mechanic: Shift+Tab cycles modes.

The loop that never fails you

Describe → Run → Observe (it broke) → Fix
You'll hit a snag. The error tells you the fix. You paste it back. That's the whole game.
💡 Breaking things isn't failure — it's the data that tells the AI what to fix.

Keep the AI sharp (the smart zone)

AI gets worse as its memory fills up — it's the guy from Memento, it forgets.
  • Keep tasks small.
  • Start fresh often.
  • A clean start is its smartest state.

Build it: "King Daryl"

  1. Open Claude Code in a fresh folder — then turn plan mode ON (Shift+Tab):
    claude
  2. Ask it, in your own words:
    Build a Chrome browser extension that replaces every "Daryl" on the page with "King Daryl". Give me the files and tell me how to load it.
  3. Load it: Chrome → Extensions → Developer mode → Load unpacked → pick the folder.
  4. Visit any page with "Daryl" and watch it work.
👑 A finished version exists — you'll see it after you build yours. The struggle-then-fix is the lesson.

Break it on purpose

This is the part everyone fears. Do it anyway:

  1. Ask for a change that might break it — e.g. also turn "SQL" into "SQL 👑".
  2. Hit the error.
  3. Paste it back.
  4. Watch Claude fix it.
Describe → Run → Observe → Fix
💡 Name the loop as you feel it: Describe → Run → Observe → Fix.

A peek at the pro workflow

For bigger projects, pros wrap this in a discipline:

grill → PRD → issues (tracer-bullet slices) → build → QA
Humans own the alignment; the agent does the work. You'll feel a light version this afternoon in M5.
