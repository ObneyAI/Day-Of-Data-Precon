What is MCP?

USB-C for AI tools

One standard plug. Any AI app can talk to any tool — no custom wiring per app.

Model Context Protocol

An open protocol that lets a Client (Claude Desktop) call Tools exposed by an MCPServer.

🔌 Before USB-C, every device had its own cable. MCP is the one connector that fits every AI app and every tool.

The Problem It Solves

💡 You write one MCPServer. Every MCP-capable Client can use it.

What an MCP Server Exposes

🎯 Today we build Tools — the actionable, "wow" part. Resources & Prompts: just know they exist.

A Tool Is Just a Function

A Tool is a function you let the AI call. The docstring + type hints are its contract — that's what the Model reads to decide when and how to use it.

@mcp.tool def semantic_search(query: str) -> list: """Find the Products most similar in meaning to `query`.""" # embed the query, rank Products by VECTOR_DISTANCE ...
💬 Write a clear docstring and the AI knows what your Tool does. That's the whole trick.

M5: Build Your Server — 4 Tools

You'll vibe-code a FastMCP server with four Tools that let Claude talk to your Catalog.

  • get_schema — tells Claude your table's columns so it writes correct SQL.
  • run_sql — runs a read-only SELECT (Claude can look, not touch).
  • semantic_search — finds products by meaning (VECTOR_DISTANCE).
  • add_product — the ONE write tool: Claude adds a product ("AI takes a real action, safely").
💡 Add them one at a time and test each as you go — vibe-code it yourself, and raise your hand if you get stuck.

Build It (1 of 2)

In Claude Code (plan mode on), describe each piece and let it build. Read what it writes; if it errors, paste the error back.

  1. Scaffold
    Create a FastMCP server in Python at mcp-server/server.py called WorkshopCatalog that runs over stdio.
  2. get_schema
    Add a tool get_schema that returns the columns and types of the Products table so the AI can write correct SQL.
  3. run_sql
    Add a read-only tool run_sql(query) that runs a single SELECT and returns rows — reject anything that isn't a read-only SELECT, and connect as the workshop_ro login so writes are blocked.
🕶 On the SQLite backup: your server connects to db/workshop_catalog.sqlite and there's no workshop_ro login — tell Claude to make run_sql reject anything that isn't a SELECT in Python instead.
🔍 Test each tool the moment you add it — in a second terminal, run the MCP Inspector, click Connect, and your new tool shows up in the Tools tab (more on that in a moment):
npx @modelcontextprotocol/inspector .venv/bin/python mcp-server/server.py

Build It (2 of 2)

  1. semantic_search
    Add a tool semantic_search(text, top_k=5) that embeds text with all-MiniLM-L6-v2 and returns the top products by VECTOR_DISTANCE cosine over the DescriptionEmbedding column.
  2. add_product
    Add a tool add_product(name, description, category, price) that inserts a product and embeds its description — the one tool allowed to write.
🕶 On the SQLite backup: SQLite has no VECTOR_DISTANCE — tell Claude to rank in Python with the cosine_distance helper in db/embeddings.py over the stored JSON embeddings. Same ranking as SQL Server.
💡 Snag → read the error → paste it back to Claude → fixed. That's the whole loop.

Test Each Tool — MCP Inspector

npx @modelcontextprotocol/inspector .venv/bin/python mcp-server/server.py
  1. Click Connect.
  2. Open the Tools tab.
  3. Each tool shows its description + parameters.
  4. The fields are pre-filled with a real example.
  5. Click Run — see live results.
💡 No AI needed yet — you're calling your tool directly. That's how easy testing is.

Wire It Into Claude Desktop

  1. Add the server to claude_desktop_config.json — command = your .venv Python, args = the full path to your server.py (use the exact snippet below).
  2. Fully quit and reopen Claude Desktop (⌘Q — not just close the window).
  3. Your Tools appear near the message box.
Mac ~/Library/Application Support/Claude/claude_desktop_config.json Windows %APPDATA%\Claude\claude_desktop_config.json
{ "mcpServers": { "WorkshopCatalog": { "command": "/full/path/to/.venv/bin/python", "args": ["/full/path/to/mcp-server/server.py"] } } }
💡 When Claude wants to use a tool it asks "Allow?" — you stay in control.

The Wow Query

"Which products are most like a rugged waterproof jacket, and what's the average price of the top 5?"
Claude reads your Tools, then: 1. calls semantic_search("rugged waterproof jacket") 2. calls run_sql("SELECT AVG(Price) ... top 5") 3. answers you in plain English

No Auth, No Cloud

Local

Client launches your server over stdio. No ports, no network calls to run it.

Read-only

run_sql takes SELECT only, backed by the workshop_ro login. The AI can look, not touch.

Yours

Your machine, your database, your Tools. Nothing leaves the laptop to make it work.

🔒 Simple on purpose. This is the safest possible on-ramp to "AI takes real actions."
